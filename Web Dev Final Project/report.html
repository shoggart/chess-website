Introduction: The essential story being told by my website is a chess news or chess blog
website. It’s a website for chess enthusiasts and newcomers alike. My website includes five
main pages and three pages that are articles about the current world of chess. The main pages
include a home page, a tutorials page for those who don’t know the rules of chess, an about
page, a chess openings page and a contact page. The main page was the most complex. It
includes a carousel at the top, flipping between three images with buttons linking to the page's
tutorials, about us and chess openings. It then contains three large images with the beginning of
an article about current news in the world of chess to entice the viewers to read more and
buttons linking to the full articles. The tutorials page walks viewers through the rules of chess
including special rules such as en passant and castling. It includes pictures of how the board is
supposed to be set up as well as some gifs on how the pieces move on the board. It also
includes a daily puzzle for viewers to try and solve. The about us page, as the name suggests,
includes information about our chess news blog. The chess openings page includes some
interactive iframe embeds that cover some of the most famous openings in chess with their back
stories and the associated algebraic notation. The contact page is a page for the viewer to
contact us at Spencer’s Chess News! Then finally the three article pages include articles on
recent chess news that I have adapted from some Chess.com articles which I’ve recently read.
Three Inspirations for the Website: The first and most obvious inspiration for this website is
that I happen to be an avid fan of chess. I play competitive bullet and blitz chess online on
Chess.com all the time. I also watch youtube videos analyzing famous chess matches and
Grand Master’s playing on twitch frequently. So when I came up with the idea of making a chess
website for my final project, I was excited to say the least. The second inspiration for this
website would have to be Chess.com. I use this website regularly and find it very easy to
navigate and enjoy many of its features. It includes lessons, many variations of different types of
chess puzzles, live online chess, tournaments, chess news, computer analysis, famous saved
chess matches, streaming by famous chess players and more. Chess.com has been an
invaluable resource to me throughout the course of developing my final project website. The
third and final inspiration for this website would have to be the excitement for learning a new
skill. I came into this class knowing absolutely nothing about web development and going from
that to developing my own site about a topic of my choice that interests me has been a fun and
challenging process.
Three Ways in which the Website is Accessible: The first way in which my website is
accessible is that it’s mobile friendly. All of the pictures and text adjust accordingly to smaller
screens and the navigation bar becomes a hamburger menu when the window is small enough.
Another way in which my chess website is accessible is via multiple different browsers. It can be
accessed from Google Chrome, Firefox, Safari and various other commonly used web
browsers. A third way that my website is accessible is through a plethora of links to it’s various
pages. There are links to the five main pages in the navigation bar, as well as links to the
tutorials page, about us page and chess openings page in the carousel. And there are also clear
links to the articles from the home page.
Three Ways which I’ve considered Usability: The first thing I did to consider website usability
was create large responsive buttons which link between the pages. This was mainly done on
the home page to link between the three articles I’ve adapted from Chess.com. A second way in
which I considered usability in the process of developing my website was by using Bootstrap 4
in order to create a flawless way to deal with mobile and desktop users. This also has the added
advantage that if another developer wants to modify or improve upon my code, they will be
operating within a framework which is familiar to them. A third way that I created a user friendly
website was in consideration of desktop users who are utilizing very large screens. I did this by
using CSS to center and set a max scale to images so images aren’t overly large on large
windows. This was particularly relevant on the home page because some of the images in the
carousel were originally blowing up to the whole size of the screen on the 32” monitor with
resolution of 2650x1440 that I was using during the development of this website.
Three Things I had to Learn: The first thing I had to learn was how to implement a responsive
website design. In order to do this, after much research, I landed upon the framework of
Bootstrap 4.0. This made creating a responsive website far easier as it was bundled with a
vendor file which contained both responsive CSS and Javascript. The second skill which I
acquired during my research for this project was the implementation of Javascript animations.
This encompassed both CSS timed animations as well as a Javascript function to implement the
carousel. The third and final lesson I have learned was how to embed an iframe within a div in
HTML. I went a little further and researched how to resize the iframe to fit for both desktop users
and mobile users.
What Worked Well: The carousel which I implemented on the home page that contained three
separate images as well as navigation control buttons which scroll through all of the images
worked very well with Bootstrap and animated smoothly. Not only that but I also implemented
text overlaid within and buttons to the aforesaid pages. Making this mobile friendly was very
straightforward as Bootstrap automatically resized the image and hid the text and buttons at a
breakpoint which could no longer properly be displayed on a small mobile screen.
What Could Be Improved: The carousel could have been touch friendly so the images can be
dragged both on desktop and mobile in order to scroll through the three images on the home
page. Another thing which could have been improved is I could have made the three article
posts on the home page scroll left and right as a carousel rather than just be static. A final thing
I could have done is create a navigational menu and/or add other widgets to the footer.
